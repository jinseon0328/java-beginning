/*
자바의 이름 짓는 규칙(naming convention)
자바는 이름을 지을 때 특정 규칙들이 존재한다.

알아들어야 할 이름 규칙
1. 낙타등 표기법(Camel back notation)
   낙타등 표기법이란, 만약 이름이 여러 단어로 이루어져 있을 경우
   두번째 단어부터는 단어의 첫글자를 대문자로 적어주는 방법이다.
   예시 : number, userNumber, MemberController
2. 뱀 표기법 (Snake noation)
   뱀 표기법이란 모든 단어를 대문자로 적어주고 여러 단어로 이루어져 있으면
   단어 사이에 _를 넣는 방법이다.
   예시 : USER_ADDR, UNIT_PRICE

기본규칙 1. 자바가 기본적으로 사용하는 50여가지의 단어들이 있다.
           해당 단어들로는 우리가 이름을 지어줄 수 없다.
           그러므로 구글링하여 해당 단어들을 모두 외워주자 --> IDE가 없던 옛날 옛적 이야기.... 외우지 말자!
           IDE란 Intergrated Development Environment의 줄임말로써
           통합 개발환경 이란 의미이다.
           IDE는 프로그램을 개발하는데 도움을 주는 여러가지 기능이 들어간 프로그램이다.
           대표적으로 Eclipse, Visual Stuio Code 등의 프로그램들이 있다.
기본규칙 2. 같은 유효범위 안에서는 똑같은 이름은 여러 개 선언해줄 수 없다.
기본규칙 3. 이름에 숫자가 포함될 수는 있지만 숫자로 이름을 시작할 수는 없다.
기본규칙 4. _를 제외한 특수문자는 사용될 수 없다.

클래스 이름 규칙: 클래스는 대문자로 시작하고 낙타등 표기법을 사용하는 명사이다.
변수 이름 규칙: 변수는 소문자로 시작하고 낙타등 표기법을 사용하는 명사이다.
메소드 이름 규칙: 메소드는 소문자로 시작하고 낙타등 표기법을 사용하는 동사이다.
상수 이름 규칙: 상수는 뱀 표기법을 사용하는 명사이다.

예전에는 이름을 지을 때 축약어를 많이 썼었다.
예시 : number일 경우 num 또는 numb
이제는 그러한 축약어를 쓰기 보다는 다 풀어서 쓰는 경우가 많아졌다.
단, 너무 길 경우에는 축약어를 쓰기도 한다.

*/
